## ✨ Возможности

### 📅 События (первая иконка в строке меню)
- **Мигает и проигрывает звук** при срабатывании оповещения о событии
- Показывает название события и обратный отсчёт до начала
- Остаётся видимым пока вы вручную не отклоните его
- **Во время события:** всегда отображается, показывает оставшееся время
- **Несколько одновременных событий:** автоматически сменяют друг друга каждые 5 секунд
- Отклонение убирает только уведомление — событие остаётся в Календаре

### ✅ Напоминания (вторая иконка в строке меню)
- Появляется когда напоминание наступило или сработал его будильник
- **Мигает и проигрывает звук** чтобы привлечь внимание
- **Требует действия** — просто отклонить нельзя:
  - **Отложить** — создаёт новый будильник через N минут
  - **Отметить выполненным** — помечает напоминание как завершённое в Напоминаниях

### ⚙️ Настройки
- Выбор календарей и списков напоминаний для отслеживания
- Включение/выключение мигания и звука; выбор звука уведомления
- Настройка вариантов длительности отложения (5, 10, 15, 30 мин и т.д.)
- Запуск при входе в систему

## 📋 Требования

- **macOS 14 Sonoma** или новее
- **Xcode 15** или новее (только для сборки из исходников)

## 📦 Установка

### Вариант 1: Скачать готовое приложение (рекомендуется)

1.  Перейдите на страницу [**Releases**](../../releases)
2.  Скачайте последний файл **CalenBar.dmg**
3.  Откройте скачанный файл и перетащите **CalenBar.app** в папку **Программы** (Applications)
4.  **Важно:** При первом запуске macOS заблокирует приложение, так как оно подписано индивидуальным сертификатом разработчика:
      - Откройте **Системные настройки** → **Конфиденциальность и защита**
      - Прокрутите вниз до раздела **Безопасность**
      - Нажмите **Подтвердить вход** (или **Open Anyway**) рядом с сообщением о CalenBar
5.  При первом запуске приложение запросит доступ к **Календарю** и **Напоминаниям** — разрешите доступ

### Вариант 2: Собрать из исходников

См. раздел [Сборка из исходников](#-сборка-из-исходников) ниже.

## 🛠 Сборка из исходников

Если вы предпочитаете собрать приложение самостоятельно:

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/just-mn/calenbar.git
cd calenbar
```

### 2. Откройте проект в Xcode

```bash
open calenbar.xcodeproj
```

### 3. Настройте подпись (опционально)

Проект настроен на **автоматическую подпись** (Automatic Signing).

- **Если у вас есть Apple ID:** Xcode автоматически создаст сертификат разработчика
- **Если нет:** Проект будет собран с ad-hoc подписью

Чтобы изменить настройки подписи:
1. Выберите проект **CalenBar** в навигаторе
2. Перейдите на вкладку **Signing & Capabilities**
3. Выберите свою **Team** или оставьте пустым для ad-hoc подписи

### 4. Соберите приложение

**Для запуска из Xcode:**
- Нажмите **⌘R** — приложение запустится сразу в строке меню

**Для установки в Программы:**
1. Нажмите **⌘B** для сборки
2. В Xcode: **Product** → **Show Build Folder in Finder**
3. Перейдите в **Products** → **Debug**
4. Скопируйте **CalenBar.app** в папку **Программы**:

```bash
# Или через терминал
cp -R ~/Library/Developer/Xcode/DerivedData/CalenBar-*/Build/Products/Debug/CalenBar.app /Applications/
```

## 🔒 Конфиденциальность

CalenBar использует системный фреймворк **EventKit** для чтения данных Календаря и Напоминаний **локально на вашем устройстве**. Никакая информация не отправляется в интернет или на сторонние серверы.

Как это работает:
- Проверяет события и напоминания каждые 30 секунд
- Мгновенно реагирует на изменения в календаре через системные уведомления
- Все данные обрабатываются только на вашем Mac

**Никакой аналитики. Никакого отслеживания. Никакого облака.**

## 🛠️ Технические детали

- **EventKit Framework** — доступ к Календарю и Напоминаниям
- **SwiftUI** — современный декларативный интерфейс
- **MenuBarExtra** — нативная интеграция со строкой меню
- Опрос каждые 30 секунд + подписка на уведомления об изменениях календаря
- Аудио через **AVFoundation** и системные звуки
- Service Management API для поддержки автозапуска

### Сообщение об ошибках

**Вариант 1: Баг-репорт из приложения (рекомендуется)**

Самый простой способ сообщить об ошибке — прямо из приложения:

1. Откройте **CalenBar** → **Настройки**
2. Перейдите на вкладку **О программе**
3. Нажмите **"Сообщить об ошибке"**
4. Нажмите **"Создать репорт"**

Приложение автоматически:
- Соберёт информацию о системе (версия macOS, версия приложения, настройки)
- Соберёт последние логи (за последние 5 минут)
- **Безопасно для конфиденциальности:** Названия событий/напоминаний и личные данные не включаются
- Откроет GitHub в браузере с заполненным шаблоном баг-репорта

Вам нужно только:
- Добавить описательный заголовок
- Описать баг
- Добавить шаги для воспроизведения
- Отправить issue

**Вариант 2: Ручное создание баг-репорта**

Если предпочитаете создать issue вручную:

1. Перейдите в [Issues](../../issues) → **New Issue**
2. Выберите шаблон **Bug Report**
3. Заполните:
   - Чёткое описание бага
   - Шаги для воспроизведения
   - Ожидаемое и фактическое поведение
   - Информацию о системе (версия macOS, версия приложения)

**Сбор логов вручную:**

При необходимости можно собрать логи из Терминала:

```bash
# Собрать логи CalenBar за последние 10 минут
log show --predicate 'subsystem == "dev.just-mn.calenbar"' \
  --last 10m --info --debug > calenbar-logs.txt
```

Логи **безопасны для конфиденциальности** — они не содержат названия событий/напоминаний или другие чувствительные данные.
